# Paste Pocket

一个适合部署到 GitHub Pages 的轻量小助手，用来统一管理：

- 常用话术
- 深度学习训练命令
- 本机本浏览器内加密保存的账号、密码、密钥

## 设计思路

这个项目刻意分成三层：

1. `公开示例`
   仓库中的 `data/library.json`，适合放不会泄露风险的示例话术和命令。

2. `本地模板`
   浏览器 `localStorage` 中明文保存，适合你自己的普通文本和命令，不会自动上传到 GitHub。

3. `加密保险箱`
   浏览器 `localStorage` 中加密保存，适合账号、密码、密钥等敏感信息。主密码只在当前会话内使用，不写入仓库。

## 安全边界

- 不要把真实账号、密码、Token、`sk` 写进 `data/library.json` 或任何仓库文件。
- 如果要接大模型自动分类，请只对非敏感内容启用。
- 如果未来一定要对敏感内容做 AI 分类，建议先本地脱敏，再通过你自己的后端代理调用模型。

## 功能

- 搜索标题、标签、正文、命令参数
- 按范围、类型、分类筛选
- 收藏与最近使用排序
- 一键复制话术、命令、账号、密码、密钥
- 新增、编辑、删除本地条目
- 创建、解锁、导入、导出本地加密保险箱

## 本地运行

因为 `fetch("./data/library.json")` 需要静态服务器，建议这样启动：

```bash
npx serve .
```

然后访问输出的本地地址。

## 部署到 GitHub Pages

1. 把整个目录推到 GitHub 仓库。
2. 在仓库设置里打开 `Pages`。
3. 选择从默认分支部署根目录。
4. 等待部署完成即可。

这是纯静态站点，不依赖数据库。

## 如果你想加 AI 分类

前端不要直接放 `sk`。推荐结构：

- GitHub Pages：前端页面
- Cloudflare Worker / Vercel Function：中转接口
- `sk`：只放在服务端环境变量

建议接口返回格式：

```json
{
  "category": "训练",
  "tags": ["深度学习", "微调", "F3"],
  "summary": "F3 数据集微调命令"
}
```

前端把结果回填到条目表单即可。
